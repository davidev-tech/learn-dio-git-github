Branches: são espécies de caminhos que contem os arquivos do projeto. São ramificações que existem dentro do mesmo repositorio.
Trabalhar com Branches nos permite trabalhar em uma ramificação do projeto sem mexer em sua estrutura principal que já está funcionando. A Branch principal costuma ter o nome de main ou master.  As ramificações da branch principal servem como uma espécie de laboratório de riscos controlados.

fork: É quando fazemos uma copia exata do repositorio para um outro repositorio, nos permitindo trabalhar no projeto sem riscos de prejudicar o repositorio principal, evitando frustrações ao usuário.

merge: O merge é a fusão de branches, quando terminamos de implementar as mudanças na branch que estamos implementando as funcionalidades ou melhorias fazemos um merge, fundindo as branches.

Não existe Branch sem commit e nem commit sem Branch.

obs: Quando criamos um repositorio remoto sem um readme ou ignore ele acaba por ser um repositorio sem branch e commit. Quando criamos com um dos dois ele já realiza um commit inicial.

head: Tag que aponta para o ultimo commit de uma branch, visualmente os commits se movimentam como uma pilha.
