# Fork

Um **fork** é uma cópia completa do repositório inteiro para **outro repositório** (geralmente na sua conta pessoal em plataformas como GitHub, GitLab etc.).

**Principais características:**
- Permite trabalhar no projeto sem risco de prejudicar o repositório original, pois você estará mexendo na sua própria cópia.
- É muito usado para contribuir com projetos de código aberto: você faz o fork do projeto original, implementa suas alterações e depois envia um **pull request** para que os mantenedores avaliem a inclusão das mudanças.
- Diferente de uma branch, o fork é uma cópia remota e independente, podendo inclusive ter suas próprias branches internamente.

**Fork vs Clone:**
- **Fork**: cópia remota do repositório (ex.: no servidor do GitHub).
- **Clone**: cópia local para a sua máquina (`git clone`).
